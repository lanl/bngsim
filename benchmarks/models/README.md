# `models/` — canonical, de-duplicated model files

Single source of truth for benchmark model files. Suites do **not** copy or
symlink models; their job manifests reference files here by path.

Layout is two-tier — **tier 1 = format, tier 2 = source**:

```
models/
  bngl/      <source>/   BioNetGen Language sources
  net/       <source>/   pre-generated reaction networks
  antimony/  <source>/   Antimony sources
  sbml/      <source>/   SBML XML
```

Some tier-2 buckets name a single provenance — `models2/` (BioNetGen
distribution), `rulehub/` (RuleHub), `pybnf/` (PyBNF examples), `ssys/`
(Antimony corpus). The remaining buckets — `bngl/{ode,ssa,psa}` + their
`net/*` counterparts, and `bngl/nf/` — hold a curated, **mixed-provenance**
corpus assembled for a suite role; **Per-model provenance** below gives the
source-per-model breakdown for those.

| Bucket | Holds | Driven by |
|--------|-------|-----------|
| `bngl/ode/`, `net/ode/`   | 47 ODE benchmark models (BNGL + pre-generated `.net`) | `suites/ode/` |
| `bngl/ssa/`, `net/ssa/`   | 12 SSA stochastic models | `suites/ssa/` |
| `bngl/psa/`, `net/psa/`   | 3 PSA models (large populations) | `suites/psa/` |
| `bngl/nf/`                | 11 network-free (NFsim) models | NFsim parity |

The `.net` files were all generated by **BNG 2.9.3**; files from BNG
2.3.1 carry `scaledUpSpecies`/`lambda` artifacts that crash
`run_network`. Regenerate with `perl BNG2.pl model.bngl`.

## Per-model provenance

Provenance for the curated `bngl/{ode,ssa,psa,nf}` buckets, cross-referenced
against the pinned upstream corpora. The `sbml/README.md` file documents the
two vendored SBML models to the same standard.

### Method & pinned sources

Each vendored file's SHA-256 was matched against three pinned upstreams
(commit + per-file SHA-256 recorded in
`parity_checks/bng_parity/manifest.json`) and the BioNetGen distribution:

| Source | Repo / distribution | Pin | License |
|--------|---------------------|-----|---------|
| RuleHub | `RuleWorld/RuleHub` | `479d6d62` | MIT |
| RuleMonkey | `richardposner/RuleMonkey` | `0f701125` | MIT |
| BNGL-Models | `wshlavacek/BNGL-Models` | `81c90d8f` | CC-BY-4.0 |
| BNG2 Models2 | BioNetGen 2.9.3 distribution | `2.9.3` | MIT (© J. R. Faeder) |

**Match** column:

- `=` — vendored file is **byte-identical** to the cited upstream copy.
- `~` — **same model**, vendored copy lightly adapted (comment/formatting/
  rate-law edits during vendoring); the cited path is the upstream of record.
- `✎` — **curated** for this benchmark: no byte/name match in a pinned
  corpus. These carry their citation and source note in the file header
  (e.g. adapted from the NFsim distribution, the BioNetGen test suite, a
  RuleBender workspace, or the Lin–Feng–Hlavacek 2019 *J. Chem. Phys.*
  stochastic-benchmark set). The **Provenance** column records the citation.

> **Note — multi-homed models.** Many canonical BNGL example models appear
> byte-identically in more than one upstream (the BioNetGen distribution's
> tutorial models are mirrored in RuleHub; the "bunching" pair below is
> byte-identical in *both* RuleHub `Published/Hlavacek2018Restructuration/`
> and BNGL-Models `my_models/ode/`; the `pybnf/` files are byte-identical to
> RuleHub's `Published/PyBioNetGen/` mirror). The table lists one upstream of
> record per file; a `=` means byte-identity to *that* copy, not exclusivity.

### Single-source buckets (context)

- `bngl/models2/` (21) — BioNetGen 2.9.3 distribution `Models2/` (canonical
  BNG example set). Several files are also mirrored byte-identically in
  RuleHub `Tutorials/` and BNGL-Models.
- `bngl/rulehub/` (7) — `RuleWorld/RuleHub` `@479d6d62`, `Published/` tree
  (`Dolan2015`, `Scaff-22_ground`, `elephant_EFA`, `example5_ground_truth`,
  `m1_ground`), all byte-identical. The `after_bunching`/`before_bunching`
  pair is byte-identical in both RuleHub `Published/Hlavacek2018Restructuration/`
  and BNGL-Models `my_models/ode/`.
- `bngl/pybnf/` (2) — PyBNF (PyBioNetGen) example set: `egfr_ground`,
  `RAFi_ground`, vendored byte-identical to RuleHub `@479d6d62`
  `Published/PyBioNetGen/core/`.
