# `bng_parity` golden references

Per-job **bngsim reference data** that downstream consumers (PyBNF,
PyBioNetGen) regenerate through their own bridges to verify they reproduce
bngsim — **without** needing a legacy-BNG / RoadRunner / AMICI install for
comparison. This is pure bngsim output: no differ, no subprocess reference side.
(The timing+parity matrix, `bng_ode_run.py` / `bng_stoch_run.py`, is what runs
legacy BNG and compares; the golden only fingerprints bngsim's own output.)

Generated by `../parity_golden.py`. Committed (unlike `runs/`, which is
gitignored). This is the first golden emitter in `parity_checks/`.

## Files

```
golden/
  golden.json            one record per swept job: checksum + fingerprint (+ optional trajectory ref)
  trajectories/<id>.json full (time, names, values) for the hand-picked subset only
```

`golden.json` uses the shared `_core` compact layout: a pretty `_meta` header
then one record per line. Read it with `_core.read_golden(path)`.

## The contract (four locked decisions)

- **D1 — which artifacts.** `.gdat` (named observables) is canonical and always
  golden when present; `.scan` (parameter_scan output) is golden alongside it
  when present; `.cdat` (species) is a **fallback** used when a job emits no
  `.gdat` and no `.scan`, *or* when its only `.gdat`/`.scan` is **degenerate**
  (e.g. a time-only `.gdat` from a model with no observables) — a network
  ODE/SSA run still writes a full species `.cdat` worth fingerprinting. (A
  network-free NFsim run writes no `.cdat`, so a no-observable NFsim job stays
  correctly skipped — there is genuinely nothing to fingerprint.) A job that
  emits several files (multi-suffix / multi-phase models — e.g. equilibration +
  main run on different time grids) is covered file-by-file: the job `checksum`
  spans **all** its files, and the `fingerprint` carries one per-file entry
  under `artifacts`.
- **D2 — stochastic basis.** A single fixed **`seed=1`** trajectory (the manifest
  `DEFAULT_SEED`), NOT an ensemble. bngsim's C++ re-seeds deterministically, so
  the same model state + `seed=1` reproduces a trajectory **exactly** — making
  the checksum a true byte-identity key. Verified: two independent `seed=1` runs
  of the same model produce identical checksums (ODE and SSA both).
- **D3 — full-trajectory subset.** Only the hand-picked allow-list in
  `parity_golden.py` (`TRAJECTORY_ALLOWLIST`, each entry with a one-line reason)
  gets its full trajectory written under `trajectories/`; every other job gets
  checksum + fingerprint only. Trajectories are rounded to the checksum's
  `checksum_sigfigs` (9) and downsampled if a file would breach the 1 MB
  `check-added-large-files` hook (`downsampled_stride` is recorded; `golden.json`
  stays full-resolution regardless).
- **D4 — mechanic.** A lean separate `parity_golden.py` consuming a bngsim
  sweep dir — not a flag on the report runner.

## Each record

```jsonc
{
  "model_id": "fast/.../akt-signaling.bngl",  // manifest key (relpath under models/)
  "method":   "ode",                          // "ode" | "stochastic"
  "checksum": "<sha256>",                      // byte-identity over ALL the job's artifacts
  "fingerprint": {
    "n_artifacts": 1,
    "kinds": ["gdat"],
    "artifacts": {                             // one flat per-file fingerprint …
      "akt-signaling.gdat": {"n_time": 251, "n_var": 6, "t_last": 500.0, "vars": {…}}
    }
  },
  "trajectory": "trajectories/…json" | null    // present only for the allow-listed subset
}
```

## How a consumer uses it

1. Regenerate the job through your bridge on the **same pinned bngsim** (match
   `_meta.versions.bngsim`) with `seed=1` for stochastic jobs.
2. Compute `_core.fingerprint.golden_pair(time, values, names)` per artifact and
   the spanning checksum the same way `parity_golden.build_golden_for_job` does.
3. **Same platform** (the intended CI case): the `checksum` must match
   byte-for-byte — the strongest check, catching any divergence.
4. **Different platform** (BLAS/rounding differs): fall back to comparing each
   `artifacts[<file>]` fingerprint with `_core.fingerprint.fingerprint_max_rel`
   against a tolerance. NOTE a single `seed=1` *stochastic* trajectory is chaotic
   — its fingerprint is **not** cross-platform-stable; the byte checksum is the
   meaningful stochastic check, and it certifies same-platform reproduction.

## BNG2.pl is required to generate

bngsim has no BNGL parser: a BNGL/.net job shells out to BNG2.pl to generate the
reaction network, then bngsim simulates it in-process. So regeneration needs
`$BNGPATH` (or `$BNG2_PL`) set even though golden never *compares* against
legacy BNG.

## Coverage

The committed set covers the **entire corpus: 895 / 895 jobs** (592 ODE + 303
stochastic `seed=1`), zero duds. Every model produces a fingerprintable
artifact.

Getting there meant *rehabilitating* the 34 jobs that originally shipped without
a runnable simulation protocol (no `actions` block, all actions commented out,
`generate_network`-only, or observables-less). Rather than diff against real
biology, each is given a short fixture that exercises bngsim and yields an
artifact — see `overrides.REHAB` (model_id-keyed; the runner strips the model's
existing actions and appends the protocol). 33 were rehabilitated; one
(`BSA_v1`) was a content-free stub (no rules/species/observables) and was
removed from the suite. **8 of the rehabbed records ride the `.cdat` fallback**
(observable-less models: a time-only `.gdat` + a full species `.cdat`), so the
D1 degenerate→`.cdat` path is exercised by real fixtures, not just asserted.

The model's directory (`my_models/{nf,ode,ssa}/`) is its *intended* method, and
the rehab matches it where the engine allows. Two unavoidable exceptions run as
ODE despite a stochastic-method directory: `cleavage_mechanism_v1` (`nf/` but no
observables → NFsim is time-only) and `agentCellSystem` (`ssa/` but bngsim's SSA
output path errors on it). The `method` field is authoritative (it drives the
oracle and the det/`seed1` output location), so the by-method counts above
reflect what actually runs.

**Crashed/timed-out runs are never fingerprinted.** A multi-action model killed
mid-write leaves a truncated trailing-action `.gdat` that is not byte-
reproducible (a consumer whose timeout lands a step earlier gets a different
file). `parity_golden` reads each unit's `_run.log` status and skips any non-`ok`
run (reason `run_<status>`). Two `glacial/` Lin2019 models (`ERK_model`,
`prion_model`) run four SSA actions each and show ~1.6–1.7× wall-time variance
(measured 292/377 s isolated, 505/708 s under load); they carry **1200 s
`timeout` overrides** so the run completes rather than boundary-races a crash.

Regenerate / widen with (NFsim glacial models are slow and must not be swept
unattended — see the suite README's memory/disk governor):

```
export BNGPATH=/path/to/BioNetGen-2.9.3
python ../parity_golden.py --regime ode --workers 4               # ODE set (fast, low risk)
python ../parity_golden.py --regime stochastic --include glacial/ --workers 2  # NFsim (supervised!)
python ../parity_golden.py --no-run-sweep --sweep-out ../runs/golden/bngsim    # rebuild json only
```
